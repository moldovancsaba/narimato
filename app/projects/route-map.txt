Route Structure for Projects:

Frontend Routes:
/projects/[slug]/page.tsx         - Display project (user-friendly URL)
/projects/edit/[id]/page.tsx      - Edit project (uses identifierMD5)
/projects/create/page.tsx         - Create new project

API Routes:
/api/projects/by-slug/[slug]      - Get project by slug (for display)
/api/projects/[id]                - Get/update project by MD5 ID (for management)
/api/projects                     - List/create projects

Example Flow:
1. View project: /projects/my-awesome-project
2. Edit same project: /projects/edit/5f3e2a7b1c9d8e4f2a1b3c4d
3. API calls:
   - View: GET /api/projects/by-slug/my-awesome-project
   - Edit: GET/PUT /api/projects/5f3e2a7b1c9d8e4f2a1b3c4d
   - Create: POST /api/projects (generates identifierMD5)
