## Objective
Record a binding product decision on vote-only mode and align code, routes, and documentation so they tell one story.

## Unified Context
Vote-only is a first-class ranking mode in the unified API (`mode: vote_only` → `VoteOnlyService` via `PlayDispatcher`). Users can start it from `pages/play.js` and `POST /api/v1/play/start`. At the same time, `RELEASE_NOTES.md` states vote-only was **removed** in v5.4.0, and `pages/vote-only.js` returns **HTTP 410 Gone** with “removed for security and compliance.”

This is not a documentation typo—it is an unresolved product fork.

## Problem
Until decided, #12, #16, and #18 cannot ship accurate docs. Operators do not know if vote-only is supported. Compliance language on the 410 page may conflict with active API surface.

## Goal
Publish `docs/adr/NNN-vote-only-status.md` choosing one path and execute the minimum code/doc changes to match.

## Scope
### In Scope
- ADR with options analysis:
  - **A. Supported:** restore `/vote-only` page or redirect to `/play?mode=vote-only`; fix RELEASE_NOTES; keep engine
  - **B. Removed:** remove `vote_only` from `PlayDispatcher`, start schema, play UI; migrate any data; 410 remains
  - **C. Deprecated:** hide UI; keep API until date X; document sunset
- Apply decision across: `README.md`, `RELEASE_NOTES.md`, `pages/vote-only.js`, `pages/play.js`, `pages/api/v1/play/vote-only/start.js`, `narimato_unified_documentation.md`

### Out of Scope
- Changing vote algorithm (VoteOnlyService vs binary search)—see #18
- Removing vote actions from swipe-more / rank-only (those are different modes)

## Execution Prompt
Draft ADR with context, decision, consequences. Share with product owner if needed. Implement the chosen path in one PR scoped to vote-only surfaces. Grep repo for `vote-only`, `vote_only`, `VoteOnly` and update every hit.

## Constraints
- If **B Removed**, ensure no README feature list item advertises vote-only
- If **A Supported**, 410 page and RELEASE_NOTES removal claims must be reverted in same release
- ADR number must be sequential in `docs/adr/`
