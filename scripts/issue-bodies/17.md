## Objective
Decide and document how `{narimato}` isolates organization data in MongoDB, then align `lib/db.js`, setup scripts, and all architecture docs with that decision.

## Unified Context
Documentation and scripts describe a **master database** plus **per-organization databases** selected via `buildOrgMongoUri()` and optional `ORGANIZATION_DB_URIS`. Runtime code uses a single `mongoose.connect(MONGODB_URI)` in `lib/db.js`. Setup scripts (`scripts/setup-databases.js`, `scripts/init-default-org.js`) create org records with `databaseName` and open second connections—but API routes never switch databases.

Tenancy today is **logical** (`organizationId` on documents), not **physical** (separate DB per org).

## Problem
Operators following setup scripts may believe data is isolated per DB when the app reads/writes one database. Security reviews will fail. Docs cannot describe `buildOrgMongoUri` until it exists or is explicitly rejected.

## Goal
Accepted ADR choosing:
- **Path A:** Implement per-org connection routing in `lib/db.js` + middleware, OR
- **Path B:** Document single-database MVP; simplify scripts; remove `databaseName` fiction from operator docs

## Scope
### In Scope
- ADR in `docs/adr/` with security, ops, and migration implications
- Update: `WARP.md`, unified doc §5, `scripts/setup-databases.js`, `scripts/init-default-org.js`
- Align `lib/models/Organization.js` with chosen schema (add `databaseName` only if Path A)
- If Path B: document that `organizationId` is the isolation boundary; indexes on `organizationId` fields

### Out of Scope
- Migrating existing Atlas data between topologies (separate migration issue if Path A)
- Row-level security beyond MongoDB app discipline

## Execution Prompt
Write ADR with decision drivers (team size, Atlas cost, compliance). If Path A, sketch `getOrgConnection(organizationId)` and list every model call site to update. If Path B, grep `databaseName` and `buildOrgMongoUri` across repo and remove or mark script-only.

## Constraints
- Path A must not ship without integration tests or scripted verification (manual script acceptable for MVP)
- Path B must not leave WARP claiming per-org DBs
- Only MongoDB Atlas `mongodb+srv://` per existing `lib/db.js` validation
