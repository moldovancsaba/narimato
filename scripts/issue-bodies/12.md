## Objective
Make `narimato_unified_documentation.md` the authoritative engineering specification for v7.2, with all companion docs deferring to it or linking to explicit ADRs.

## Unified Context
Today five documents disagree on fundamentals: Next.js version (15.4.4 / 15.5.3 / 15.5.9), UUID storage (`OrganizationUUID` literals vs `organizationId` in Mongo), API base paths, and play-mode count. The **code** uses Pages Router JavaScript, `lib/constants/fieldNames.js` as a logical→physical field map, and `/api/v1/play/*` plus `/api/cards*`.

Without one canonical spec, every other issue in this program re-opens the same arguments.

## Problem
`ARCHITECTURE.md` still describes TypeScript files under `app/lib/models/Organization.ts`, a `Session` model with optimistic locking, and `useOrganizationTheme`—none of which exist. `narimato_unified_documentation.md` claims to be canonical but lists v6.9.0 and Next 15.4.4. Contributors cannot tell which file to trust.

## Goal
Publish an updated unified spec at **7.2.0** that matches `package.json`, `lib/models/*`, `pages/api/*`, and `PlayDispatcher.js`, and relocate aspirational content to `docs/FUTURE.md`.
