## Objective
Update `README.md` so every listed feature, API path, and play-mode claim is true in v7.2 or explicitly marked as planned in `docs/FUTURE.md`.

## Unified Context
README is the first document evaluators and contributors read. It currently markets **full dark mode**, **optimistic locking with conflict resolution**, **binary search** as the primary ranking mechanism, and **six play modes**—while the codebase has no theme system, no `version` field on `Play`, uses `VoteOnlyService` on the v1 vote path, and exposes **seven** modes including `onboarding` in `pages/api/v1/play/start.js`.

README also implies all modes use only the unified API; `pages/play.js` still calls legacy `/api/play/*` for classic and hierarchical flows.

## Problem
False README claims create legal/expectation risk (“full dark mode”), wasted implementation time, and failed demos when features do not exist.

## Goal
A README that accurately represents v7.2: what works today, what is deprecated, and what is future.
