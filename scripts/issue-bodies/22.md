## Objective
Delete macOS duplicate `* 2.js` files under `lib/` that are not imported anywhere, after confirming no unique fixes exist only in duplicates.

## Unified Context
The repo contains **35** files matching `* 2.js` (e.g. `Card 2.js`, `SwipeOnlyPlay 2.js`, `VoteOnlyService 2.js`). `require()` grep shows **zero** production imports of these paths. Some duplicates **diverge** from canonical files—e.g. `SwipeOnlyPlay 2.js` omits `onboardingIndex` that `SwipeOnlyEngine.js` requires on the canonical model.

These files confuse audits, agents, and editors (“which Card model is real?”).

## Problem
Duplicate modules create false documentation targets and risk accidental edits on the wrong file. Drift between pairs can mislead future merges.

## Goal
Zero `* 2.js` under `lib/`; canonical files retain any intentional changes discovered during diff review.

## Scope
### In Scope
- Diff each `* 2.js` against its non-suffixed twin
- Port any intentional-only changes into canonical file (if any)
- Delete all `* 2.js` under `lib/`
- Run `npm run build` and `npm run lint`
- Add `* 2.js` to `.gitignore` or editor note in CONTRIBUTING if duplicates recur (optional)

### Out of Scope

- Removing root-level `debug-*.js` scripts (separate cleanup)
