## Objective
Rewrite `docs/API_REFERENCE.md` so every documented HTTP endpoint exists in `pages/api/` with accurate request and response shapes.

## Unified Context
The unified Play API (`POST /api/v1/play/start`, `input`, `next`, `results`) is implemented and used by `pages/play.js` for modern modes. The API reference, however, still documents a pre-migration play start (`cardName`, `sessionId`, `playUuid`) and a non-existent Cards API at `/api/v1/cards` with `body.textContent` and `operation: SAVE|GENERATE|IMGONLY`.

The real cards surface is `/api/cards`, `/api/cards/[uuid]`, and `/api/cards/rankings`ŌĆöused by `pages/cards.js` and `pages/results.js` but undocumented in the trio of top-level docs.

## Problem
API consumers and agents following `API_REFERENCE.md` will call wrong URLs and send wrong payloads. This is a **critical** integration failure, not a wording issue.

## Goal
Produce an API reference where:
1. Unified v1 play section matches `pages/api/v1/play/*.js` and `PlayDispatcher.js`
2. Cards section documents `/api/cards*`
3. Legacy `/api/play/*` routes are in a clearly labeled **Deprecated** chapter (see #19)
4. Historical spec is archived, not presented as current
